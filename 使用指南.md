# 蓝牙控制器 APP - 手机使用指南

## 前提条件

1. 一台 Android 手机（Android 6.0 及以上）
2. 一台电脑（用于安装和打包）
3. 数据线（连接手机和电脑）
4. 手机蓝牙已开启

---

## 方法一：通过数据线安装（推荐）

### 步骤 1：在电脑上准备项目

1. 打开命令行/终端，进入项目目录：
   ```bash
   cd c:/Users/admin/WorkBuddy/20260318104350
   ```

2. 安装依赖（首次运行需要）：
   ```bash
   npm install
   ```
   这可能需要几分钟，请耐心等待。

### 步骤 2：设置手机开发者模式

**在手机上操作：**

1. 打开「设置」→「关于手机」
2. 连续点击「版本号」7次，直到提示「您已处于开发者模式」
3. 返回「设置」，找到「系统」或「更多设置」
4. 进入「开发者选项」
5. 开启「USB 调试」

### 步骤 3：连接手机

1. 用数据线连接手机和电脑
2. 手机上会弹出「允许 USB 调试」的提示，点击「允许」或「始终允许」

### 步骤 4：安装到手机

在电脑上运行：
```bash
npm run android
```

等待几分钟，APP 会自动安装到手机上并启动。

---

## 方法二：生成 APK 安装包安装

如果不想每次都连电脑，可以生成安装包（.apk 文件）：

### 步骤 1：进入 Android 目录

```bash
cd android
```

### 步骤 2：生成 Release APK

Windows:
```bash
gradlew.bat assembleRelease
```

Mac/Linux:
```bash
./gradlew assembleRelease
```

### 步骤 3：找到 APK 文件

APK 文件位置：
```
android/app/build/outputs/apk/release/app-release.apk
```

### 步骤 4：传输到手机安装

1. 将 `app-release.apk` 复制到手机（通过数据线、微信发送、或云盘）
2. 在手机上找到这个文件并点击安装
3. 如果提示「未知来源」，需要在「设置」→「安全」中允许安装未知应用

## 如何使用 APP

### 1. 首次启动

打开 APP 后，会弹出权限请求：
- ✅ 附近设备（蓝牙扫描）
- ✅ 发现并连接到附近的设备（蓝牙连接）
- ✅ 获取精确位置（Android 12 以下需要）

**点击「允许」全部授权**

### 2. 扫描设备

1. 确保手机蓝牙已开启
2. 点击「🔍 扫描设备」按钮
3. 等待 10 秒，会显示发现的蓝牙设备列表

### 3. 连接设备

1. 在设备列表中，点击你要连接的设备
2. 等待连接成功，状态会显示「✅ 已连接」
3. 控制按钮区域会出现

### 4. 发送控制指令

连接成功后，点击以下按钮发送字符：

| 按钮 | 发送字符 |
|------|---------|
| 🟦 W | 发送 `w` |
| 🟩 A | 发送 `a` |
| 🟧 S | 发送 `s` |
| 🟥 D | 发送 `d` |
| 🟪 Q | 发送 `q` |
| 🟦 F | 发送 `f` |

每次发送会在底部记录中显示。

### 5. 断开连接

点击「🔌 断开连接」即可断开当前蓝牙设备。

## 常见问题

### Q: 扫描不到设备？
- 检查目标设备是否已开启并处于可被发现状态
- 确保手机蓝牙已开启
- 尝试关闭其他 APP 中的蓝牙连接

### Q: 连接失败？
- 确保目标设备支持 BLE（蓝牙低功耗）
- 目标设备可能已被其他设备连接
- 尝试重启蓝牙或手机

### Q: 安装 APK 提示「解析包错误」？
- 确保 APK 文件完整下载
- 可能是 Android 版本不兼容（需要 Android 6.0 及以上）

### Q: 如何卸载？
- 长按 APP 图标，选择「卸载」
- 或在「设置」→「应用」中找到「蓝牙控制器」卸载

## 目标设备要求

要被此 APP 控制的设备需要满足：

1. 支持 BLE（蓝牙低功耗）4.0 及以上
2. 有可写的特征值（Characteristic）
3. 能够接收单字节字符数据

常见可用设备：
- Arduino + BLE 模块（如 HM-10）
- ESP32（内置蓝牙）
- 各种 BLE 开发板
- 支持 BLE 的智能硬件
